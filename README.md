# Go Microservices Example

Проект, демонстрирующий построение **production‑ready** веб‑сервисов на чистой стандартной библиотеке Go.  
Включает два независимых компонента, которые могут работать как вместе, так и по отдельности:

- **In‑memory кэш с TTL и REST API** – конкурентно‑безопасное хранилище с автоматической очисткой.
- **HTTP‑клиент для GitHub API** – с автоматическими повторами (retry), обработкой rate limit и поддержкой контекста.

---

## 📦 Особенности

### Общие
- **Чистая стандартная библиотека** – без внешних веб‑фреймворков (Gin, Echo и т.п.).
- **Конкурентная безопасность** – мьютексы, атомарные операции, защита от гонок.
- **Graceful Shutdown** – корректное завершение при получении SIGINT/SIGTERM.
- **Логирование** – каждый запрос и повторная попытка логируются с деталями.
- **Тестирование** – юнит‑ и интеграционные тесты с `httptest`, проверка на гонки (`-race`).

### Кэш (REST API)
- **REST API** – управление кэшем через HTTP (GET, POST, DELETE, очистка).
- **Автоматическая очистка** – фоновая горутина удаляет истёкшие записи каждую секунду.
- **Валидация TTL** – проверка формата (`time.ParseDuration`) и положительности.
- **Таймауты** – настроены `ReadTimeout`, `WriteTimeout`, `IdleTimeout`.
- **Middleware** – логирование запросов и восстановление после паник.

### GitHub клиент
- **Метод `GetUser`** – получение информации о пользователе GitHub.
- **Автоматические повторы (retry)** – при ошибках 5xx, сетевых сбоях и 429 (Too Many Requests).
- **Экспоненциальная задержка** – от `MinDelay` до `MaxDelay` (по умолчанию 1–30 секунд).
- **Обработка rate limit** – чтение заголовков `X-RateLimit-*` и ожидание при исчерпании лимита.
- **Поддержка контекста** – отмена запроса и установка дедлайнов.
- **Логирование** – все запросы, повторные попытки и ошибки.

## 🚀 Быстрый старт

### Требования
- Go 1.22+ (используется `r.PathValue()`)

### Установка и запуск сервера

```bash
# Клонируем репозиторий
git clone https://github.com/rezect/ttl-cache-server.git
cd ttl-cache-server

# Скачиваем зависимости (только для тестов)
go mod download

# Запускаем сервер (порт по умолчанию :8000)
go run cmd/server/main.go

# Или с указанием порта через переменную окружения
PORT=8080 go run cmd/server/main.go
```

Сервер запустится, и вы сможете отправлять запросы на `http://localhost:8000` (или указанный порт).

## 📚 API (кэш)

Все запросы и ответы – в формате JSON.

| Метод | Путь | Тело запроса | Описание | Успешный ответ | Ошибки |
|-------|------|--------------|----------|----------------|--------|
| **GET** | `/cache/{key}` | – | Получить значение по ключу | `200 OK` `{"value": <any>}` | `404 Not Found` (ключ не найден или истёк) |
| **POST** | `/cache/{key}` | `{"value": <any>, "ttl": "5s"}` | Создать или обновить ключ с TTL (обязательно) | `200 OK` `{"status": "ok"}` | `400 Bad Request` (невалидный JSON, TTL отсутствует или <=0) |
| **DELETE** | `/cache/{key}` | – | Удалить ключ | `204 No Content` (если ключ был) <br> `404 Not Found` (если не было) | – |
| **POST** | `/cache/clear` | – | Очистить весь кэш | `200 OK` `{"status": "ok"}` | – |
| **GET** | `/health` | – | Проверка работоспособности | `200 OK` `{"health": "Ok"}` | – |

### Примеры запросов

```bash
# Создать/обновить ключ
curl -X POST http://localhost:8000/cache/user \
  -H "Content-Type: application/json" \
  -d '{"value": {"name": "Alice", "age": 30}, "ttl": "1m"}'

# Получить значение
curl http://localhost:8000/cache/user
# => {"value":{"age":30,"name":"Alice"}}

# Удалить ключ
curl -X DELETE http://localhost:8000/cache/user

# Очистить весь кэш
curl -X POST http://localhost:8000/cache/clear

# Проверка здоровья
curl http://localhost:8000/health
# => {"health":"Ok"}
```

## 🌐 GitHub API Client

Пакет `githubclient` предоставляет HTTP‑клиент для взаимодействия с GitHub API.

### Пример использования

```go
package main

import (
    "context"
    "log"
    "time"
    "github.com/rezect/ttl-cache-server/pkg/githubclient"
)

func main() {
    cfg := githubclient.DefaultConfig()
    cfg.BaseURL = "https://api.github.com/users"
    cfg.Timeout = 10 * time.Second
    cfg.MaxRetries = 3
    cfg.MinDelay = 1 * time.Second
    cfg.MaxDelay = 30 * time.Second

    client := githubclient.NewClient(cfg)

    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()

    user, err := client.GetUser(ctx, "torvalds")
    if err != nil {
        log.Fatalf("Error: %v", err)
    }
    log.Printf("User: %+v", user)
}
```

### Тестирование клиента

Клиент покрыт тестами с использованием `httptest.NewServer`, эмулирующего GitHub API:

| Сценарий | Описание |
|----------|----------|
| **200 OK** | Успешный ответ с данными пользователя. |
| **404 Not Found** | Проверка обработки ошибки (без retry). |
| **500 Internal Server Error** | Проверка механизма retry. |
| **429 Too Many Requests** | Проверка ожидания по заголовку `X-RateLimit-Reset`. |
| **Отмена контекста** | Проверка, что запрос прерывается при отмене контекста. |

Запуск тестов клиента:

```bash
go test -race ./pkg/githubclient/...
```

## 🔗 Взаимодействие кэша и GitHub клиента

Кэш может использоваться для хранения результатов запросов к GitHub API, чтобы снизить количество внешних вызовов и избежать rate limit.

**Пример архитектуры:**
1. Сервер получает запрос на `/github/user/{username}`.
2. Проверяет кэш – если данные есть и не истекли, возвращает их.
3. Если данных нет – делает запрос через `githubclient`, сохраняет результат в кэш (с TTL, например, 1 час) и возвращает ответ.

Этот подход демонстрирует **комбинирование компонентов** для решения реальной задачи – кэширования внешних данных.

## ⚙️ Конфигурация

### Сервер
Переменная окружения:
- **`PORT`** – порт для сервера (по умолчанию `8000`).

Таймауты сервера (захардкожены):
- `ReadTimeout`  – 5 секунд
- `WriteTimeout` – 10 секунд
- `IdleTimeout`  – 120 секунд

### GitHub клиент
Параметры конфигурации (структура `Config`):

| Поле | Тип | Описание | Значение по умолчанию |
|------|-----|----------|----------------------|
| `BaseURL` | `string` | Базовый URL API | `https://api.github.com/users` |
| `Timeout` | `time.Duration` | Таймаут на весь запрос | `30s` |
| `MaxRetries` | `int` | Максимальное число повторных попыток | `3` |
| `MinDelay` | `time.Duration` | Минимальная задержка между попытками | `1s` |
| `MaxDelay` | `time.Duration` | Максимальная задержка | `30s` |
| `Logger` | `*log.Logger` | Логгер для вывода | `log.Default()` |
